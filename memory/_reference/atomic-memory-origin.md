# 原子記憶系統（Atomic Memory System）

## 起源 — 為什麼需要它？

大模型 AI 工具（AntiGravity、Codex、Claude Code 等）在實際開發中，共同面臨一個根本問題：

> **跨 Session 失憶。**

具體而言：

- 每次開新對話，AI 對先前的工作一無所知
- 即便是**重複的簡單動作**，仍需使用者重新描述——而 AI 依然可能做錯
- 錯誤一旦發生，容易引發連鎖反應：反覆修整、越改越歪
- 更遑論讓 AI 對中大型專案產生**持續且實質**的助益

---

## 構想 — 從人類認知到設計靈感

### 人腦的「偷懶」智慧

人類面對物理硬限制（大腦能耗有限），演化出一套極其務實的反應——**偷懶**。

人類記憶的本質是：

```
印象 + 加權
```

大腦長期處於一種「**能耗精簡**」狀態：

> 能用記憶覆蓋的答案，就不重新推導。

這從大量密集使用腦力時的熱量消耗現象即可得證——思考是昂貴的，所以大腦本能地節省。

### LLM 的困境

然而對 LLM 而言，**一切知識是平等的**。

即便透過上下文關鍵字檢索，粒度已細緻到「單字」層級（如：*A 的下一個是 B*），每一次推理仍需消耗算力，沒有「印象」可以走捷徑。

### 阿賴耶識的啟發

這種差異，讓我聯想到佛教所說的「**阿賴耶識**」——

一切經驗的種子皆儲存於此，不需全部浮現於意識表面，卻能在適當時機被喚起。

---

## 核心設計哲學

> **我們不要清清楚楚地知道一切，**
> **而是「印象派抽取記憶法」。**

並非要完整復刻佛教教義——LLM 究竟能達到什麼程度，仍因環境而異。

但我們可以透過兩個關鍵手段：

| 手段 | 說明 |
|------|------|
| **串接** | 跨 Session 的記憶持續累積與連結 |
| **縮小範圍** | 從 LLM 宏大的知識中，精準聚焦到當下需要的部分 |

以此達成——**類似人類的短、中、長程記憶能力**。

---

## 延伸 — 從記憶到職人

後續補增的大量工具，實質上就像人類靠著各種能力來補強記憶：

- **習慣** — 自動化的工作流程與規則
- **技能** — 專門化的處理工具與腳本
- **職人直覺** — 從經驗萃取的模式識別與決策支援

記憶系統不只是「記住」，而是讓 AI 具備**經驗累積**的能力，達成更優質的輔助開發效果。
