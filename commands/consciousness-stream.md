# 識流處理（Consciousness Stream Pipeline）

> 以佛教八識體系的結構化管線處理任務：感知→分析→校準→執行→迴寫。
> 適用於高風險跨系統任務，或需要嚴謹分析的場景。

---

## 使用方式

```
/consciousness-stream <任務描述>
```

### 參數

| 參數 | 必填 | 說明 | 範例 |
|------|------|------|------|
| 任務描述 | 是 | 要處理的任務（自然語言） | `重構 ModuleQuery 的反射呼叫機制` |

### 使用範例

```
/consciousness-stream 分析 MapServer Handler 的錯誤處理覆蓋率
/consciousness-stream 重構 ModuleQuery 的反射呼叫機制
/consciousness-stream 評估 Observer 洩漏的修復方案
```

### 錯誤處理

- **未帶任務描述** → 顯示本使用說明，結束
- **任務描述過於模糊**（如「改程式」）→ 詢問使用者補充具體範圍

---

## 任務

$ARGUMENTS

## Step 0: 觸（Sparśa）— 風險分級

用以下關鍵詞規則判定風險，**不需額外推理**：

| 等級 | 關鍵詞 |
|------|--------|
| **極高** | 框架、基類、共用定義、breaking、base class、shared |
| **高** | 修改、重構、核心、刪除、遷移、跨系統、refactor、delete、migrate、core |
| **中** | 新增、建立、設定、配置、create、add、config、setup |
| **低** | 讀取、搜尋、分析、查看、列出、報告、read、search、analyze、list（預設） |

- 低風險 → **簡化模式**（跳過 Step 1 中的五識/八識/光明心部分）
- 中以上 → **完整模式**

輸出：任務摘要（一句話）、風險等級、涉及領域、模式

## Step 1: 感知 + 記憶收集

### 五識感知（完整模式才全做）
- **眼識**: 讀取任務涉及的檔案
- **鼻識**: `git status`、環境狀態
- **舌識/身識**: 程式碼品質、系統資源（視需要）

### 八識種子庫（完整模式）
- 讀取 `_AIDocs/_INDEX.md`，查找相關文件
- 檢索相關 atoms（按 Trigger 欄位比對）
- 檢索 pitfalls、changelog

## Step 2: 分析（六識 + 七識合併）

**一次完成以下所有項目：**

### 三量分別
- **現量**（pratyakṣa）：直接觀察到的事實，不加推測
- **比量**（anumāna）：基於事實的推論，標明依據
- **非量**：不確定的假設，明確標記

### 四煩惱自檢
- **我癡**: 有不知道但假裝知道的嗎？未讀原始碼就下結論？
- **我見**: 執著特定技術而忽略替代方案？
- **我慢**: 高估本地能力、低估外部資源？
- **我愛**: 偏袒自己之前的作品？

### 交叉比對
- 與 `_AIDocs` 既有知識比對
- 確認既有決策（decisions.md）是否相關

輸出：校正後的判斷摘要

## Step 3: 轉智 + 校準（光明心 + 四智 + 執行規劃合併）

**一次完成以下所有項目：**

### 光明心 — 原則校準（完整模式）
- 輕量極簡？ 高可讀性？ 薄框架層？ 安全合規？
- **他空見**：不空（善用本地能力）vs 空（放下過度依賴）

### 四智行動計畫
| 智 | 內容 |
|----|------|
| 成所作智 | 具體行動清單：做什麼、改哪些檔案、步驟順序 |
| 妙觀察智 | 為什麼這樣做、分析依據 |
| 平等性智 | 對相關模組/平台的影響評估 |
| 大圓鏡智 | 與系統整體架構的一致性 |

### 執行 Checkpoint
- 每個步驟的成功條件

## Step 4: 執行

按行動計畫逐步執行，每步確認 checkpoint。
- 結果符合預期 → 繼續
- 發現新資訊 → 記錄，待 Step 5 薰習
- 需要調整 → 回到 Step 3 重新校準

## Step 5: 薰習（Vāsanā）— 經驗迴寫

規則化提取，更新記憶系統：
- 非量中含「坑」「問題」「風險」→ 寫入 pitfalls
- 四煩惱自檢非 null 項 → 寫入 decisions [臨]
- `_CHANGELOG.md` 追加
- MEMORY.md 更新（若為高頻/重要資訊）

## 輸出格式

```markdown
### 觸 — 任務辨識
- 本質 / 風險 / 涉及 / 模式

### 六識 — 分別分析
**現量：** / **比量：** / **非量：** / **交叉比對：**

### 七識 — 自省校正
**四煩惱自檢：** / **校正判斷：**

### 八識 — 種子檢索
### 光明心 — 原則校準
### 四智 — 行動計畫
| 智 | 內容 |
### 薰習 — 經驗迴寫
```
