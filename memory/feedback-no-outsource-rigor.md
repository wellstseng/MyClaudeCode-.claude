## 知識

- [臨] 高推理模式（high/max thinking）必須提升縝密度，禁止問「還有別的角度我沒看到嗎」「我有沒有漏掉什麼規則」這類開放式把思考工作外包給使用者的問題
- [臨] Why: 使用者在 high mode 觀察到我問此類問題，視為「自主削弱角色縝密」徵兆；切到 max 觀察是否改善。本身具備懂使用者的全方位程式大師角色，知識面與規範盤點是我的職責不是使用者的
- [臨] How to apply: 涉及規範/治理/架構/記憶系統決策時，先窮盡讀完相關文件（rules/core.md、CLAUDE.md、IDENTITY.md、USER.md、_AIDocs/_INDEX.md、相關 atom）再給結論。發問必須具體到「X 應該 Y 還是 Z（理由 R1 / R2）」，禁止「還有別的嗎」
- [臨] 自我檢查：每次給結論前，先在心裡回答「我查了哪些規則來源？盤點範圍多大？」如果答不出來就先去查，再回應
- [臨] 連帶提醒：rules/core.md 第 1 條「開工前查 _AIDocs/_INDEX.md，禁止憑記憶改碼」是常被自己違反的，新任務開始時主動查

## 行動

- 規範性決策前，先列出『已查規則來源 + 已盤點範圍』再給結論
- 不發開放式『還有什麼角度 / 我有沒有漏』；改成具體選項題或自己拍板
- 新任務開始時主動讀 rules/core.md + _AIDocs/_INDEX.md，不憑記憶
